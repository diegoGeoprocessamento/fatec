---PROJETO PARA SEGUINTES RESOLUÇÕES

320 pixel – Smartphones no formato retrato
480 pixel – Smartphones no formato paisagem
768 pixel – Tablets no formato retrato
960 pixel – Tablets no formato paisagem e alguns monitores mais antigos
1200 pixel – Desktops com monitores widescreen

---Estrutura de trabalho
estilo.css
index.html
folder imagens

Deve-se analisar o layout e identificar as seções para criar as divs, tags e classes.

Background - Imagem distribuída para todo o layout.
Header - div #topo todo o cabeçalho contido na tag <header> logotipo e menu.
Seções/Conteúdo - Seção .chamada, .container com 3 colunas .desktop, .tablet e .mobile.
Footer - com duas colunas para os conteúdos.

<meta name="viewport" content="width=divice-width, initial-scale=1.0">
A maioria dos navagadores para dispostivos móveis, tentam exibir as páginas
como se estivessem sendo exibidas em desktops, ocacionando problemas de responsividade.

A meta tag viewport trabalha com resoluções personaçlizadas inclusive na rendereziação do zoom,
manipulando a escala incial do site.

---Configurações básicas css

utilizar o normalize.css
http://necolas.github.io/normalize.css/

<head>
    <title>Como Criar Site Responsivo com HTML5 e CSS3 - CarlosHPS Blog</title>
    <meta charset="utf-8">
    <link rel="stylesheet" type="text/css" href="estilo.css">
    <link href='http://necolas.github.io/normalize.css/+Sans:300,400,600,700,800' rel='stylesheet' type='text/css'>
    <meta name="viewport" content="width=device-width, initial-scale=1.0">
</head>
